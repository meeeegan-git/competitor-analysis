# 服饰运动素材爆款榜单 - 产品需求文档（PRD）

> 版本：v1  
> 日期：2026-02-27  
> 产品定位：**对外展示给客户的数据报告**  

---

## 一、产品概述

### 1.1 产品目标
基于每周导出的广告投放数据（CSV），生成一个可直接访问的**静态网页数据报告**，面向客户展示服饰运动品类的素材爆款榜单。页面需具备专业的视觉设计、丰富的交互体验和数据可视化能力。

### 1.2 核心用户
- 品牌方/广告主（查看行业投放趋势）
- 内部运营团队（选品与素材分析参考）

### 1.3 数据源
每周提供一份 CSV 文件，包含以下 16 个字段：

| 序号 | 字段名 | 说明 |
|------|--------|------|
| 1 | KPI三级行业 | 行业分类，如"女装"、"运动鞋服-品牌"等（新增） |
| 2 | 商品统一类目V2(一级～四级) | 四级类目路径 |
| 3 | 素材MD5示意(预览) | 素材标识 |
| 4 | 素材MD5示意(预览)(翻译后) | 素材视频MP4链接 |
| 5 | 消耗(元) | 广告消耗金额 |
| 6 | 消耗(元)占比(%) | 消耗占比 |
| 7 | 下单金额(元) | 下单总金额 |
| 8 | 下单金额(元)占比(%) | 下单金额占比 |
| 9 | 下单单价(元) | 客单价 |
| 10 | ctr(%) | 点击率 |
| 11 | 综合目标转化率(%) | 转化率 |
| 12 | 竞价CPM(元) | 千次展示成本 |
| 13 | 下单ROI | 投产比 |
| 14 | 视频3秒完播率(%) | 视频前3秒完播率 |
| 15 | 平均播放时长(毫秒精度)(s) | 视频平均播放时长 |

## 二、数据处理逻辑

### 2.1 去重规则
- 按 **DPA商品名称** 进行去重
- 同一商品名称下有多条素材时，**只保留消耗最大的那条素材**

### 2.2 取数规则
- 按"商品统一类目V2(一级～四级)"分组
- 每个类目内，去重后按消耗降序，取 **TOP 100** 条素材

### 2.3 数据更新机制
- 每周由运营提供新的 CSV 文件
- 通过预处理脚本（`node scripts/process-csv.mjs <csv路径> <周标签>`）生成 JSON 数据
- 历史周数据持久保存，支持按周切换查看

## 三、页面结构与功能

### 3.1 整体布局

```
┌─────────────────────────────────────────────────┐
│  顶部导航栏（Logo + 标题 + 周期选择器）           │
├─────────────────────────────────────────────────┤
│  数据概览仪表盘（统计卡片 + 雷达图）               │
├─────────────────────────────────────────────────┤
│  筛选工具栏（类目筛选 + 单价范围 + 搜索）          │
├─────────────────────────────────────────────────┤
│  商品榜单列表（电商卡片风格，内嵌视频播放器）       │
├─────────────────────────────────────────────────┤
│  分页 / 加载更多                                  │
└─────────────────────────────────────────────────┘
```

### 3.2 顶部导航栏
- **Logo + 标题**："服饰运动素材爆款榜单"
- **周期选择器**：
  - 下拉选择不同周（如 2026-W09、2026-W10）
  - 切换周后页面数据自动刷新
  - 默认展示最新一周数据

### 3.3 数据概览区

~~#### 3.3.1 统计卡片~~ **已移除** - 不展示总素材数、类目数量等统计卡片
- 仅在筛选工具栏右侧显示"共 X 条素材"的简要数量提示

#### 3.3.2 雷达图（各类目核心指标对比）
- 展示**已选类目**（或默认消耗TOP5类目）的核心指标对比
- 雷达图维度：消耗(元)、下单ROI、CTR(%)、3秒完播率(%)、平均播放时长(s)
- 各维度数据需归一化处理（0-100分制）
- 支持切换对比的类目

### 3.4 筛选工具栏

#### 3.4.1 行业筛选（原类目筛选已替换）
- 使用 TDesign `Select` 下拉搜索+多选组件
- 筛选字段：**KPI三级行业**（网页中显示为"行业"）
- 用户输入关键词即可模糊搜索，选中后以折叠标签形式显示
- 支持清除全部已选行业
- 共14个行业：女装、男装、鞋类、箱包、贴身衣物、运动鞋服-品牌/非品牌 等

#### ~~3.4.2 下单单价范围筛选~~ **已移除**

#### 3.4.3 排序
- 默认按消耗绝对值降序（**前端不展示消耗数值**，仅用于排序）
- 用户无需手动切换排序

### 3.5 商品榜单列表（核心区域）

#### 3.5.1 展示风格
采用**类电商平台商品列表**布局，每个商品为一张**横向卡片**，包含：

```
┌─────────────────────────────────────────────────────────────┐
│  ┌──────────┐                                               │
│  │          │  #1  商品名称（粗体，单行截断带tooltip）         │
│  │  视频    │  类目标签（浅色Tag）                            │
│  │  播放器  │                                                │
│  │          │  ┌─────┐ ┌──────┐ ┌─────┐ ┌──────┐            │
│  │  悬停    │  │日耗  │ │下单   │ │ROI  │ │CTR   │  ...       │
│  │  自动播放│  │区间  │ │单价   │ │     │ │      │            │
│  │          │  └─────┘ └──────┘ └─────┘ └──────┘            │
│  └──────────┘                                               │
│  素材分析：#前3秒亮点 #人物形象 #卖点 #脚本结构               │
└─────────────────────────────────────────────────────────────┘
```

#### 3.5.2 每张卡片包含的字段

| 字段 | 展示方式 |
|------|----------|
| 排名 | 左上角序号徽章（TOP3高亮金/银/铜色） |
| 商品主图 | 80x80px 区域，**使用素材视频第2秒画面作为缩略图**（`<video src="...#t=2" preload="metadata">`），悬停自动播放、移出暂停并回到第2秒 |
| 商品名称 | DPA商品名称，最多2行截断带tooltip |
| 类目标签 | 名称下方浅灰色显示四级类目路径 |
| 行业 | KPI三级行业，浅蓝色Tag展示 |
| ~~日耗区间~~ | **已移除** - 不再展示消耗相关数据 |
| 下单单价 | 单位：元 |
| 下单ROI | 颜色编码：≥2绿色、1~2黄色、<1红色 |
| 视频3秒完播率(%) | 百分比展示 |
| 平均播放时长 | 单位：秒 |
| CTR | 百分比展示 |
| 素材分析 | 标签化展示（由AI生成，当前显示"待AI生成"） |
| 素材链接 | 播放图标+链接，点击新窗口打开视频 |

#### 3.5.3 分页
- 每页 20 条
- 底部分页器，支持页码跳转

### 3.6 素材分析（AI生成）
- 基于素材视频链接，由 AI 自动分析生成
- 分析维度：
  1. **前3秒亮点**：视频开头的吸引点
  2. **人物形象**：出镜人物特征
  3. **卖点**：产品核心卖点
  4. **素材脚本结构**：视频整体叙事结构
- 输出格式：标签化（每个维度一个标签），总字数控制在50字以内
- **注意**：AI分析需要调用视频分析能力，如暂不可用则该字段显示为空

## 四、视觉设计要求

### 4.1 设计风格
- **对客报告级别**：专业、简洁、高端感
- 配色方案：深蓝为主色调，搭配白色/浅灰背景，数据高亮使用品牌色
- 卡片设计：圆角、轻阴影、hover微交互
- 字体：中文使用 PingFang SC / 微软雅黑，数字使用 DIN / Tabular Nums

### 4.2 响应式
- 主要针对桌面端优化（1200px+）
- 平板端可访问但非重点

## 五、技术方案

### 5.1 技术栈
- **前端**：React + TypeScript + Vite
- **UI组件库**：TDesign React
- **图表库**：Recharts（雷达图等）
- **样式**：Tailwind CSS
- **数据**：构建时内嵌 JSON（由预处理脚本从CSV生成）

### 5.2 数据流
```
CSV文件 → process-csv.mjs脚本 → JSON数据文件 → Vite构建 → 静态网页
```

### 5.3 每周更新流程
1. 运营提供新的CSV文件
2. 执行 `node scripts/process-csv.mjs <csv路径> <周标签>`
3. 执行 `npm run build`
4. 部署到静态托管
